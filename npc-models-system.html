EnsureColumns() — Otomatik Migrasyon
Ne Yapar?

Sunucu açılışında tablet_gangs tablosunda npc_models kolonu var mı kontrol eder. Yoksa ALTER TABLE ile ekler. Manuel SQL gerektirmez.

MySQL.query("SHOW COLUMNS FROM `tablet_gangs` LIKE 'npc_models'", ...) -- Kolon yoksa: ALTER TABLE tablet_gangs ADD COLUMN npc_models TEXT DEFAULT NULL
Buraya JSON array olarak model adları kaydedilir:
["g_m_y_ballaeast_01", "s_m_y_dealer_01"]
İki Yeni Fonksiyon

İşlem

Kullanım

03 — server/bandana.lua

Gang Güncelleme
Handler'ları

İki ayrı updateGang handler'ı mevcut (satır 519 ve 670). Her ikisine de aynı blok eklendi.

Neden İki Handler?

Kodun farklı zamanlarda yazılmış eski/yeni versiyonu. FiveM'de aynı event iki kez kayıt edilince sadece son kayıt edilen çalışır — ama hangisinin "son" olduğu load order'a göre değişir. İkisi de güncellendi.

Eklenen Blok
if data.npcModels ~= nil then local models = type(data.npcModels) == 'table' and data.npcModels or {} DB.UpdateGangNpcModels(data.gangId, models, function() if Territory_RefreshGangNpcModels then Territory_RefreshGangNpcModels(data.gangId, models) end end) end
Geriye Dönük Uyumluluk

data.npcModels ~= nil kontrolü: eski event'ler npcModels göndermezse mevcut modeller silinmez.

Admin'den Servere Akış

fetchNui('updateGang', { ..., npcModels: ["model1"] })

DB.UpdateGang()
label, color, icon güncellenir.

DB.UpdateGangNpcModels()
npc_models JSON olarak tablet_gangs'a kaydedilir.

Territory_RefreshGangNpcModels()
O gang'ın tüm territory'lerine canlı broadcast gider.
04 — server/admin.lua

Admin Veri
Parse Katmanı

Admin paneli açıldığında tüm veriyi frontend'e gönderen Admin_SendData() fonksiyonuna npc_models JSON parse bloğu eklendi.

Parse Bloğu
for _, g in ipairs(gangs) do -- Mevcut ranks parse (değişmedi) if type(g.ranks) == 'string' then local ok, p = pcall(json.decode, g.ranks) g.ranks = ok and p or {} end -- YENİ: npc_models parse if type(g.npc_models) == 'string' then local ok, p = pcall(json.decode, g.npc_models) g.npc_models = ok and p or {} end end
Frontend'e Giden Gang Objesi

pcall Güvencesi

DB'de bozuk JSON varsa pcall sayesinde sunucu crash olmaz, boş array döner.

Yardımcı Fonksiyonlar
🔧
_loadModel(hash)

_spawnOnePed() — Ana Spawn Fonksiyonu
1
Model seç (round-robin)
3 model varsa → 1,2,3,1,2,3... sırayla. ((idx-1) % #models) + 1
2
Hash al ve yükle
GetHashKey(modelName) → _loadModel(hash) — geçersiz model atlanır.
3
Rastgele pozisyon
Zone merkezi etrafında açı + mesafe hesabı. Z ekseni için GetGroundZ kullanılır.
4
CreatePed(4, ...)
Tip 4 = civilian. Gang görünümü için model seçimi admin'e bırakıldı.
5
Davranış ayarla
Kaçmaz, patlama tepkisi yok, TaskWanderStandard 25m yarıçapında dolaşır.
Ana Thread Mantığı — Her 5 Saniyede
🔄
StartNpcThread() — Periyodik Kontrol
CheckIntervalMs: 5000
1
Oyuncu pozisyonunu al
GetEntityCoords(PlayerPedId())
2
Tüm TerritoryNpcData'yı döngüyle tara
npc_models dolu mu? → 2D mesafe hesabı (dx²+dy² ≤ r² — sqrt almaktan kaçın)
3
Zone bulunduysa EnterZone() çağır
Önceki zone değiştiyse eski ped'leri sil, yeni zone tracking başlat
4
Ped listesini temizle ve say
_cleanDeadPeds → alive = _countAlivePeds → needed = MaxPerTerritory - alive
5
Needed kadar yeni ped spawn et
_spawnOnePed() döngüsü. Ölmüş/silinen ped'lerin yerine yenisi konur.
6
Hiç zone bulunamadıysa ExitZone()
Önceden zone'daydık → tüm ekstra ped'leri DeleteEntity ile sil.
Sync Event Handler'ları
📡

➕

Yeni territory claim edildi. TerritoryNpcData'ya ekle — thread sonraki turda yakalar.
🗑️

TerritoryNpcData'dan sil. İçindeysek ExitZone() çağır, ped'leri temizle.
🔄

Model güncellendi → ZoneExtraPeds sıfırla → Thread yenilerini spawn edecek.
